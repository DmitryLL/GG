# Геймдизайн-данные

JSON-конфиги. Сервер при старте подтягивает их через
`nakama-server/scripts/gen-data.js` → `src/data.gen.ts` → бандл.

## Файлы

| Файл | Что | Когда менять |
|---|---|---|
| `items.json` | Каталог предметов: оружие, броня, расходники, материалы. | Добавил новое оружие/броню; меняешь цены или урон/HP. |
| `mobs.json` | Типы мобов: HP, скорость, урон по касанию, XP/gold за убийство, скорость респауна. | Балансишь сложность; добавляешь нового моба. |
| `drops.json` | Таблицы дропа на тип моба. `weight` — относительный вес лута, `missWeight` — пустой дроп. | Меняешь шансы выпадения. |
| `npcs.json` | NPC на карте: торговцы и их сток. | Перенёс торговца, открыл нового. |
| `balance.json` | Глобальные числа: атака игрока, дальность подбора, размер инвентаря, формула XP. | Тонкая настройка ощущений. |

## Чтобы добавить нового моба

1. В `mobs.json` добавь блок типа:
   ```json
   "wolf": {
     "hpMax": 45, "touchDamage": 8, "speed": 70,
     "wanderRadius": 100, "touchRange": 28,
     "touchCooldownMs": 800, "respawnMs": 6000,
     "xp": 18, "gold": 2
   }
   ```
2. В `drops.json` добавь его дроп-таблицу (можно пустую `{ "missWeight": 100, "table": [] }`).
3. В клиенте — спрайт `godot/assets/sprites/mobs/wolf.png` (32×32, 2 кадра по горизонтали как у slime/goblin).
4. Прописать в `godot/scripts/mob.gd` (массив `def`) — путь к текстуре и масштаб. (Эту часть пока надо менять кодом — позже вынесем.)
5. В `world.json` (когда сделаем) укажи где спавнятся волки. Сейчас спавны генерируются процедурно от seed.

## Чтобы изменить цену предмета

`items.json` → найди нужный → поменяй `price` (покупка у NPC) или `sellPrice` (продажа NPC).

## Деплой

`git push` — CI запустит сборку, Nakama перезапустится с новыми данными. Без редеплоя пока не работает (это в фазе «hot reload» админки).
