# godot/data — data-driven параметры

Эта папка для JSON/`.tres`-файлов, которые **читаются кодом при запуске**, но которые удобно править без знания GDScript. Идея: всё, что Вова может захотеть крутить (визуальные коэффициенты, баланс, имена), вынесено сюда; логика остаётся в `godot/scripts/`.

Владелец: **@protiik** (Вова).

## Что тут лежит

| Файл | Что описывает | Читает код |
|---|---|---|
| `mobs.json` | Визуал мобов: текстура, масштаб, кол-во кадров, FPS анимации | `scripts/mob.gd` |

## Правила

1. **Менять только значения**, не ломать структуру (ключи). Если нужно новое поле — согласовать с Димой.
2. **Путь к ассетам** — `res://assets/sprites/...`. Относительные пути не работают.
3. **Новый моб**: добавить ключ в `mobs.json` + сам спрайт в `assets/sprites/mobs/NEWNAME.png`. После этого серверу (Dima) нужно добавить тип моба в `nakama-server/src/`, иначе спавнить нечего.
4. **Ошибка в JSON** ломает клиент при запуске (файл не распарсится). Проверяй через jsonlint.com или просто запуск Godot.

## Как добавить новый тип моба (пример: волк)

1. Положи спрайт: `assets/sprites/mobs/wolf.png` (32×32 пикс, hframes=N, vframes=1).
2. В `mobs.json`:
   ```json
   "wolf": {
     "texture": "res://assets/sprites/mobs/wolf.png",
     "scale": 1.35,
     "frames": 4,
     "fps": 5.0
   }
   ```
3. ПР, тег Диме. Дима добавит серверный спавн.
