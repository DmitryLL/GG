# Godot assets — правила для визуала

Тут живут все спрайты, анимации, тайлсеты, звуки и карты. Владелец: **@protiik** (Вова). Логика (`godot/scripts/`) и «скелетные» сцены (`godot/scenes/core/`) зоны Димы — их менять отсюда не надо.

## Структура

```
assets/
├── sprites/
│   ├── characters/    игроки, экипировка (char_*.png, wear/)
│   ├── mobs/          враги (goblin, slime, dummy, butterflies, npc)
│   ├── projectiles/   стрелы, снаряды (arrow.png, arrow_crit.png)
│   ├── skills/        иконки скиллов (skill_1..5.png, class_archer)
│   ├── items/         предметы инвентаря (items.png, boulder)
│   └── ui/            иконки UI (icon_bag, icon_stats, auth_bg, tiles)
├── animations/        SpriteFrames .tres — по одному на персонажа/моба
├── tilesets/          наборы тайлов для TileMap
├── maps/              Tiled-карты (world.tmj)
└── audio/             звуки и музыка
```

## Контракт анимаций

Чтобы скрипты Димы не ломались при переделке визуала, любой `AnimatedSprite2D` / `SpriteFrames` должен иметь фиксированный набор анимаций по именам:

### Персонаж (игрок и человекоподобные NPC)

| Имя анимации | Когда играет | Зацикленность |
|---|---|---|
| `idle`        | стоит без действия      | loop |
| `walk`        | движение                | loop |
| `attack`     | базовая атака           | одноразово |
| `cast_1` … `cast_5` | каст скиллов 1..5 | одноразово |
| `hurt`        | получил урон            | одноразово |
| `death`       | смерть                  | одноразово (последний кадр держится) |

### Мобы

| Имя | Когда |
|---|---|
| `idle`   | стоит |
| `walk`   | движется |
| `attack` | атакует |
| `hurt`   | получил урон |
| `death`  | смерть |

### Снаряды (стрелы, шары)

| Имя | Когда |
|---|---|
| `fly`    | в полёте (loop) |
| `impact` | попадание (одноразово → удаляется) |

## Правила

1. **Имена анимаций фиксированы.** Если нужно новое — согласовать с Димой и добавить сюда.
2. **Визуальные префабы лежат в `scenes/visual/`**, а не в `scenes/core/`. Там шаблоны вида `PlayerVisual.tscn`, `MobGoblinVisual.tscn` — только спрайт + AnimationPlayer, без логики.
3. **Не трогаем `scripts/` и `scenes/core/`** — это зона Димы. Если для новой анимации нужно изменение кода, открываем ПР с тегом `@DmitryLL`.
4. **Большие файлы** (> 1 МБ) — предварительно прогнать через сжатие (pngquant / tinypng). Git хранит все версии, раздувается быстро.
5. **Спрайт-листы** именовать `{subject}_{action}.png`: `char_archer_walk.png`, `goblin_attack.png`.

## Как добавить нового моба

1. Кладём спрайт-лист в `sprites/mobs/newmob_walk.png` (и остальные действия).
2. Создаём `animations/mob_newmob.tres` — SpriteFrames с анимациями по контракту.
3. Создаём `scenes/visual/MobNewmobVisual.tscn` — `AnimatedSprite2D`, подхватывает .tres.
4. Пингуем Диму — он подключит визуал в логическую сцену моба.
