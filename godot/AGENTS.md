## 🎬 Контракт анимаций (ВАЖНО)

Коду Димы нужны анимации с **фиксированными именами**. Если сделаешь префаб с другими именами — игра не увидит анимацию и персонаж будет замороженный.

### Для персонажей (`PlayerVisual.tscn` и подобных)

| Имя | Когда играет | Loop? |
|---|---|---|
| `idle`           | стоит без действия     | ✅ |
| `walk`           | движется               | ✅ |
| `attack`         | базовая атака          | ❌ |
| `cast_1` … `cast_5` | каст скиллов 1..5   | ❌ |
| `hurt`           | получил урон           | ❌ |
| `death`          | смерть (последний кадр держится) | ❌ |

### Для мобов (`MobGoblinVisual.tscn` и подобных)

`idle`, `walk`, `attack`, `hurt`, `death`

### Для снарядов (`ArrowVisual.tscn`)

`fly` (loop), `impact` (oneshot)

Если нужно новое имя анимации — **сначала попроси Диму добавить вызов** в соответствующем скрипте, только потом делай анимацию.

## 🧩 Правила создания визуальных префабов

Корневой узел префаба должен называться **`Visual`** и быть `Node2D`. Внутри — `AnimatedSprite2D` (или `Sprite2D` + `AnimationPlayer`).

**Пример `PlayerVisual.tscn`:**
```
Visual (Node2D)
└── AnimatedSprite2D
    └── SpriteFrames с анимациями: idle, walk, attack, cast_1..5, hurt, death
```

**Чего нельзя:**
- Подключать скрипт `.gd` к префабу.
- Использовать пути за пределами `res://assets/...`.
- Называть файл не по шаблону: используй `<Что>Visual.tscn` в PascalCase.
- Добавлять коллайдеры, триггеры, `Area2D` — это логика, не визуал.

## 🎨 Как добавить новый спрайт

1. Кладёшь `.png` в правильную подпапку `assets/sprites/<категория>/`.
2. Если это анимация персонажа/моба — дополни или создай `SpriteFrames` в `scenes/visual/`.
3. Если нужно, чтобы код сразу начал его использовать — **проверь, есть ли путь в коде**:
   - Мобы: путь в `godot/data/mobs.json` — можешь менять сам.
   - Иконки UI / скиллов / персонажа: путь может быть захардкожен в скрипте. Если файл переименован — нужна правка в `.gd`. Это единственный случай, когда Вова трогает `scripts/`. Лучше сначала заменить файл с тем же именем (без правок кода) — самый безопасный путь.

---

## 🐉 Как добавить нового моба

Пример: хочешь добавить волка.

1. **Спрайт:** `assets/sprites/mobs/wolf.png` (формат как у `goblin.png` — 32×N пикс, hframes=число кадров, vframes=1).
2. **Data:** добавь ключ в `godot/data/mobs.json`:
   ```json
   "wolf": {
     "texture": "res://assets/sprites/mobs/wolf.png",
     "scale": 1.35,
     "frames": 4,
     "fps": 5.0
   }
   ```
3. **Тег Диму в PR** — ему нужно добавить тип `wolf` в серверный код (`nakama-server/src/`), иначе сервер не будет спавнить таких мобов.

## 🧪 Как проверить, что ничего не сломал

1. Открой проект в Godot 4.3+ (`godot/project.godot`).
2. F5 — запуск. Смотри консоль на **красные ошибки**.
3. Пройди базовый путь: вход → логин → мир → постреляй по мобам → скиллы → инвентарь → выход.
4. Если что-то красное про «missing texture» или «file not found» — ты переименовал файл, но не обновил ссылку. Найди в `scripts/` или `scenes/` старый путь и либо восстанови файл, либо попроси Диму обновить ссылку.

## 🌲 Git: как работать

**Ветка Вовы:** `vovan` (или заводи фичеветки `feature/vova-<что>`).

```bash
git checkout vovan
git pull origin main       # всегда подтягивай main перед работой
# … правки …
git add godot/assets/sprites/mobs/wolf.png godot/data/mobs.json
git commit -m "Мобы: волк (спрайт + параметры)"
git push origin vovan
gh pr create --base main --head vovan --title "Мобы: волк"
```

**Правила:**
- В коммит кладёшь **только свои файлы** — ассеты, визуальные префабы, JSON из `data/`. Если в `git status` виден файл из `scripts/` или `scenes/core/` — ты что-то случайно задел, откати.
- Коммит-сообщения по-русски, кратко по сути.
- Не push в `main` напрямую — только через PR.
- Никогда не делай `git push --force` или `git reset --hard` без явного согласия.

## ⚠️ Чего нельзя делать никогда

1. **Не упоминай название референс-игры** (той самой, на которую похож проект) — ни в коде, ни в коммитах, ни в документах.
2. **Не ставь скрипты `.gd` на префабы из `scenes/visual/`.** Эти сцены — чистый визуал.
3. **Не удаляй и не переименовывай** файлы из `scripts/`, `scenes/core/`, `project.godot`, `.github/`.
4. **Не меняй структуру ключей** в `data/*.json` (только значения). Если нужна новая структура — ПР с тегом Димы.
5. **Не коммить секреты**: `export_credentials.cfg`, `.env`, любые API-ключи.
6. **Не ставь большие бинари** (>1 МБ) без сжатия. PNG прогоняй через pngquant.
